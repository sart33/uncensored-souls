1. General Information

⚠️ 18+ — When you launch the app for the first time, you will see an age restriction screen. If you are under 18 — press "No".

2. API Keys Setup

Go to: Settings → API Keys

You need to enter two keys:

DeepSeek API (required for chat)

DeepSeek API Keys platform.deepseek.com/api_keys

Minimum deposit: $2. If you set a response limit, this can last for a fairly long time.

Novita AI (for image generation)

Novita AI API Keys novita.ai/dashboard/api-keys

Minimum deposit: $10. One image costs about half a cent, so this is enough for a large number of generations.

  • DeepSeek key entered → chat becomes available
  • Novita key entered → image generation is unlocked
  • API keys are stored only on your device

3. Quick Setup

Chat Text Size

After entering API keys, go to settings and choose a comfortable chat font size.

5. Multi-Chat

In the Multi-Chat section you will see two default characters — Natasha and Anya. You can start chatting immediately. Each character responds independently.

  • Tap a character icon → open their gallery
  • Use Edit to add characters, change behavior, or edit the greeting message
  • Optimal: 2–4 characters
  • Maximum: up to 5 — more may result in overly long responses or character confusion

5.1 Fixing Response Attribution Issues

In long chats with many characters, some may occasionally go silent or respond as if they were a different character. If this happens, add an explicit response format instruction to the Multi-Chat behavior card:

Reply in turns. Format strictly as:
CharacterName: 'text', CharacterName: 'text'
💡 This forces the model to clearly label each response and maintain separate voices. It is usually not necessary, but in long sessions with 4–5 characters it reliably prevents mixing and silence.

6. Chat Usage

Click a character to open their chat list. If no chat exists — create a new one.

6.1 General Flow

After sending a message, response time is 2–30 seconds. Multi-chat may take longer.

6.2 Controlling Responses

You can specify desired response length.

  • Continue → continues the scene
  • More Details → enables detailed mode (actions, emotions, reactions of all participants)

6.3 Image Generation in Chat

In single-character chats, a Photo button is available. It visualizes the current scene — showing the character as they appear right now, in the current setting, with the actions and mood of the ongoing conversation.

How it works

When tapped, the app automatically extracts the scene:

  • Current location
  • Character's actions, pose, and mood
  • Recent chat context

This allows you to see the character in a café, trying on clothes, on a walk, in the kitchen, and many other situations. Scene extraction works correctly in 54 languages — on rare languages, location may be slightly less accurate, but character actions are generally recognized well.

Limitations
  • Physical contact (hugging, kissing, dancing together, etc.) is not supported — since the app doesn't know what the user looks like, the character will be generated alone. They may, for example, blow a kiss or stand nearby.
  • Explicit sexual acts are not directly depicted. The generator produces arousing poses, suggestions, or invitations to intimacy — "before" or "after" scenes, but not the act itself.
  • Well-supported scenes: walks, cafés, kitchen, trying on clothes, lingerie, nudity in various settings, and other everyday or intimate moments.
Working with results
  • If the first image doesn't quite match your vision — tap Regenerate. The next generation may better reflect what you had in mind.
  • Tap the image to open it full-screen. If the current branch contains multiple images, you can swipe through them as a gallery.
Tips for better results

The more precisely the scene is described in chat, the better the output. Specify location, clothing, time of day, mood, and actions. You don't need to describe yourself — just describe what the character is doing and where they are.

  • With little detail — the model will improvise more
  • With sufficient detail — you'll likely get exactly what you expected

Even after the first greeting message you can get a decent image, but the richer the context, the more accurate the result.

Image storage and cleanup
🔒 All generated images are stored inside the app and do not appear in the device gallery. If images accumulate over time, go to Settings → Auto-delete chat images to enable automatic cleanup (7 to 60 days). This applies only to chat images — the character gallery is not affected.

6.4 Message Editing

Long press on a message to edit, delete, or copy. After editing or deleting, the conversation continues from that point.

6.5 Voice Input (Android)

Voice input via keyboard is recommended. The model generally understands context well, but it's better to review the text before sending.

6.6 Long Chats and Summarization

Branches

Don't hesitate to create new branches and forks. You can edit both your own messages and the character's. The last character message can be regenerated at any time.

Automatic summarization

Every 30 messages (by default), the app generates a summary of key events — a compressed "memory block". This serves two purposes:

  • Saves tokens — without summarization, longer chats become progressively more expensive
  • Keeps the character aware of your shared history — name, relationship, key events — even as the conversation grows

Each new 30-message block adds another summary. The number of active summary blocks loaded into context is configurable:

SettingEffect
0No memory, maximum token savings
4 (default)Balanced — covers recent shared history
HigherCharacter remembers older events, but costs more tokens

In practice, 4 blocks is sufficient for most long-term conversations.

Character stability
  • Personality Reminder — enabled by default (every 10 messages). Increasing frequency (e.g. every 5) improves character consistency.
  • Temperature (Creativity) — recommended range: 0.7–0.75. Lower → more stable and predictable. Higher → more creative, but higher risk of personality drift.
Response length

You can manually limit the model's response length in settings if you prefer short, concise replies. DeepSeek adapts well to any communication style — brief exchanges work just as well as detailed ones.

7. Data Export and Import

Uncensored Souls uses a local-first architecture — all your data is stored on your device only. To back up your data or transfer it to another device (Android ↔ Windows ↔ macOS), use the export/import feature.

What is transferred

  • All characters and their settings
  • All chat branches (including your messages)
  • Character avatars
  • Character gallery images
  • Images generated inside chats

7.1 Creating a Backup (Export)

  1. Go to Settings → Export and Import Data
  2. Tap "Export Data"
  3. Wait for the process to complete
✅ A green notification will confirm: "Data exported successfully."
File name format: uncensored_souls_backup_20260510.usbackup

7.2 Importing Data

  1. In the same section, tap "Import Data"
  2. Select a .usbackup file via the system file manager
  3. A warning will appear: "Existing characters with matching IDs will be updated."
  4. Tap "Confirm"
  5. Wait for the import to complete (spinning indicator)

Once done, all data (characters, chats, images) will be immediately available in the app.

7.3 Platform-Specific Notes

Android
  • Export automatically saves to the Downloads folder
  • The file can be easily copied or shared via Telegram, email, USB, etc.
Windows
  • On export, you choose the save folder manually
  • On import, the standard Windows Explorer opens
macOS — Alpha version
  • Export saves the file via the standard Finder dialog
  • On import, the file is first placed in the app's internal folder, which then opens automatically with the file highlighted
  • You can copy it to a convenient location (e.g. Desktop or iCloud) for transfer between devices

7.4 Tips

  • A single .usbackup file contains everything — only one file needs to be transferred
  • The file can be opened as a regular zip archive (in WinRAR, enable "show all file types")
  • It is recommended to export before major app updates or when switching to a new device
  • Import updates existing characters with the same ID rather than creating duplicates

8. Monitoring Your Spending

The current version does not include a built-in expense tracker. However, you can monitor all spending accurately and in detail directly on the providers' websites.

8.1 Chat (DeepSeek)

DeepSeek Usage Dashboard platform.deepseek.com/usage
  • Daily statistics: tokens used and amount spent in cents
  • Chat accounts for the vast majority of spending — typically 90–98% of total costs
  • Character verification and scene extraction costs are negligible

8.2 Image Generation (Novita AI)

Novita AI Billing Details novita.ai/billing/details
  • Shows current balance and daily spending breakdown
  • One generation or regeneration costs approximately $0.005 (~0.5¢)
  • If a generation fails (e.g. due to server load), no charge is applied
